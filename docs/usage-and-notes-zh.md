# Audio_Typewriter 项目使用与注意事项（中文）

本文档用于日常使用与交接，重点说明启动方式、核心流程、以及高风险操作的后果。

## 1. 启动前准备

- 建议先执行一次环境初始化（Python 依赖 + Web 依赖）。
- 确认 `ffmpeg`、`ffprobe` 可用（已加入 PATH 或在环境变量中指定）。
- 如果你要用 MFA 对齐：
  - 先自行网上搜索并完成 `Montreal Forced Aligner (MFA)` 安装。
  - 安装后把本机路径填写到 `start.bat` 顶部的 `AT_ASR_MFA_*` 变量示例位置。

## 2. 推荐启动方式

在项目根目录执行：

```powershell
Set-Location "E:\Audio_Typewriter"
.\start.bat
```

说明：
- `start.bat` 会先拉起后端，再等待健康检查后拉起前端。
- 若你未配置完整 MFA 路径，系统会自动降级为非 MFA 对齐流程，不会阻塞启动。

## 3. 音频库导入流程（当前主流程）

1. 在前端输入 `base_name`（音频库名）与本地文件夹路径。
2. 选择界面语言（中文/英文）；导入时该语言会传入 ASR 任务。
3. 后端读取本地目录中的 `.wav/.mp3`，进行 VAD + ASR，并写入数据库索引。
4. 处理完成后可用于混音与词库导出。

## 4. 中文识别与词库存储规则

- 当导入任务语言为 `zh` 时：
  - ASR 按中文流程处理并入库。
  - 词库导出会保留中文词/字，不再被英文默认语言覆盖。
- 当导入任务语言为 `en` 时：
  - 走英文流程并输出英文词项。

## 5. 混音与片段输出

支持：
- 拼接模式：`word` / `word_phrase` / `word_phrase_sentence`
- 输出模式：
  - `mix`：输出完整拼接句子
  - `segment_output`：按片段输出多个 wav（用于人工后剪）

提示：
- `segment_output` 默认片段扩充为 `250ms`。
- 若只做人工剪辑，优先用 `segment_output` 可减少自动拼接误差影响。

## 6. 词库导出

- 当前“导出词库”按钮只导出当前选中音频库。
- 导出内容为词项列表（不含时间戳、出现次数）。

## 7. 高风险操作与后果（务必阅读）

### 7.1 覆写同名音频库

当你导入一个已存在的 `base_name`，系统会进行“覆写”逻辑：

- 清理该音频库的历史索引与库内音频文件。
- 丢弃该音频库未完成任务（等待中/运行中）。

这意味着：旧数据不可用于恢复（除非你事先做了备份）。

### 7.2 删除任务（破坏性操作）

在任务列表点击“删除任务”后，会清理该任务对应 `base_name` 的相关内容，包括但不限于：

- 任务队列中的相关任务条目
- 数据库里的音频源与词索引
- `audio_base` 下对应库文件
- 相关临时目录（例如 VAD 作业目录）

这不是“仅删一条任务记录”，而是对该音频库的整套关联数据清理。

### 7.3 reASR

- `reASR` 会先清理该库现有 ASR 索引，再重新排队转写。
- 若中断或失败，需在任务面板恢复任务继续执行。

## 8. 隐私与仓库管理建议

建议将以下内容视为敏感信息，避免提交到仓库：

- 本机用户名路径（如 `C:\Users\<name>\...`）
- 私有模型目录、个人磁盘路径
- 任务队列运行快照（可能包含人名/项目名）
- API Key、Token、私钥文件

建议实践：

- 本地路径统一使用占位示例，不写真实用户名路径。
- 运行态文件（如任务队列 json）尽量不纳入版本控制。
- 发布前执行一次全仓库敏感信息扫描。

## 9. 常见问题

### 9.1 为什么前端能开但后端报 MFA 警告？

- 说明你启用了 `mfa/auto` 对齐但 `AT_ASR_MFA_*` 未配置完整。
- 不影响基础可用性，系统会回退到非 MFA。

### 9.2 为什么导入后词库不是中文？

- 检查导入时语言是否选择为 `zh`。
- 检查该任务是否在语言修复后新建；旧任务可能仍按旧参数执行。

### 9.3 删除任务后数据还能恢复吗？

- 默认不能。请在删除前自行备份数据库与 `audio_base` 目录。
